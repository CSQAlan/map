# 助老地图手机现场采集手册

适用对象：不写代码、不碰数据库的队友。  
目标：拿手机到重庆师范大学试点区，把路段适老数据采集回来，供管理员审核后进入正式路线算法。

## 1. 二维码怎么让手机打开系统

二维码本质上不是一个特殊 App，它只是把一个网页地址编码成图片。

例如电脑上启动前端后，页面地址可能是：

```text
http://192.168.1.23:5173
```

把这个地址生成二维码后，队友用手机相机或微信扫一扫，手机浏览器就会打开这个网址。

## 2. 为什么手机能访问电脑上的网页

手机能打开系统，需要满足一个条件：手机和电脑在同一个网络里。

最常见方式：

- 电脑连接校园 Wi-Fi，手机也连接同一个校园 Wi-Fi。
- 或者手机开热点，电脑连手机热点。
- 或者电脑开热点，手机连电脑热点。

这时电脑会有一个局域网 IP，比如：

```text
192.168.1.23
```

前端页面跑在电脑的 `5173` 端口，后端接口跑在电脑的 `8000` 端口，所以手机访问的是：

```text
http://电脑局域网IP:5173
```

如果手机和电脑不在同一个网络，手机通常打不开这个地址。

## 3. 比赛或现场采集前怎么准备二维码

由负责技术的同学在电脑上完成：

1. 启动数据库、后端和前端。
2. 查看电脑局域网 IP。
3. 确认手机能打开 `http://电脑局域网IP:5173`。
4. 用任意二维码生成工具把这个地址生成二维码。
5. 把二维码发到群里，或打印出来。

Windows 查看 IP 可以用：

```powershell
ipconfig
```

找当前 Wi-Fi 或热点对应的 `IPv4 地址`。

## 4. 现场队友怎么使用

队友只需要按这个流程：

1. 手机连接指定 Wi-Fi 或热点。
2. 扫二维码，打开助老地图网页。
3. 点击顶部的“采集模式”。
4. 在“选择采集路段”里选择当前正在看的路段。
5. 点击“记录手机定位”。
6. 按现场情况填写坡度、路宽、台阶、坡道、扶手、路面、安全、无障碍、休息设施等字段。
7. 在备注里写清楚特殊情况。
8. 提交为待审核记录。

队友不用安装 Python、Node、PostgreSQL，也不用知道数据库是什么。

## 5. 手机真实定位到底需不需要

需要，但只作为辅助证据，不作为唯一依据。

原因：

- 手机 GPS 在校园楼宇、树荫、窄路附近会有误差。
- 常见误差可能是 5 到 20 米。
- 我们采集的是某一小段路是否有台阶、坡道、扶手，这种信息靠 GPS 自动判断不可靠。

所以第一版采用：

```text
人工选择路段 + 手机定位辅助 + 备注说明 + 管理员审核
```

定位的作用是证明“采集人当时大概在现场附近”。正式更新哪条路段，仍然以“选择的路段”和“审核结果”为准。

## 6. 这样采集靠谱吗

靠谱，但前提是按规则采，不把它当成随手填问卷。

我们用三重机制保证可靠性：

- 第一重：队友现场观察和测量。
- 第二重：定位、备注、照片作为证据。
- 第三重：管理员审核后才覆盖正式路段数据。

也就是说，现场提交的数据不会立刻影响算法。它会先进入待审核列表，确认无误后再更新 `road_segment`，路线推荐才会使用新数据。

## 7. 每条路段至少采什么

每条路段至少记录：

- 路段名称。
- 路宽。
- 坡度。
- 台阶数量。
- 台阶高度。
- 是否有坡道。
- 是否有扶手。
- 是否轮椅可通行。
- 路面类型。
- 路面平整度 1-5。
- 安全等级 1-5。
- 无障碍等级 1-5。
- 休息设施评分 1-5。
- 照明等级 1-5。
- 过街安全等级 1-5。
- 树荫覆盖比例。
- 座椅数量。
- 现场备注。

如果后面接入照片上传，每条路段建议至少拍两张：

- 从起点看向终点。
- 从终点看向起点。

如果有台阶、坡道、破损路面、扶手，也要单独拍细节。

## 8. 非计算机队友怎么判断 1-5 分

### 路面平整度

- 5 分：非常平整，轮椅和老人都很好走。
- 4 分：基本平整，只有少量裂缝或小起伏。
- 3 分：能走，但明显有不平、砖缝或小坑。
- 2 分：破损、坑洼、碎石明显，老人走有风险。
- 1 分：不建议老人通行。

### 安全等级

- 5 分：人车分离，视野好，无明显危险。
- 4 分：基本安全，偶尔需要注意车辆或人流。
- 3 分：需要明显注意车辆、人流或转弯盲区。
- 2 分：有较明显安全隐患。
- 1 分：高风险路段。

### 无障碍等级

- 5 分：轮椅可以稳定通行，无台阶或有良好坡道。
- 4 分：轮椅基本可通行，但需要谨慎。
- 3 分：拐杖老人可通行，轮椅通行有压力。
- 2 分：有明显障碍，比如台阶、窄路、坡道不友好。
- 1 分：行动不便老人不建议通行。

### 休息设施评分

- 5 分：有座椅、树荫或可停靠休息点。
- 4 分：附近有休息条件，但不算特别方便。
- 3 分：可以短暂停留，但没有明确设施。
- 2 分：几乎没有休息点。
- 1 分：长距离无休息条件。

## 9. 现场采集分工建议

建议两人一组：

- A 同学负责走路段、观察、测量。
- B 同学负责手机填写、拍照、备注。

每采完一条路段，两个人现场互相确认一次：

```text
路段选对了吗？
有没有台阶？
轮椅能不能过？
坡道和扶手有没有？
备注有没有写清楚？
```

这样比一个人边走边填更稳。

## 10. 常见问题

### 手机打不开二维码页面

先检查：

- 手机和电脑是不是同一个 Wi-Fi 或热点。
- 地址里的 IP 是不是电脑当前 IP。
- 前端服务是否正在运行。
- Windows 防火墙是否阻止访问。

### 定位失败怎么办

可以继续提交，但备注里写：

```text
手机定位失败，已人工确认位于该路段现场。
```

定位失败不等于数据无效，因为我们还有人工选择路段和管理员审核。

### 不知道怎么填坡度怎么办

优先用手机水平仪或坡度 App。没有工具时，可以先粗略估计，并在备注里写：

```text
坡度为目测估计，后续建议复测。
```

### 路段列表里找不到当前路

不要随便选一个相似路段。备注记录当前道路位置，回去后由技术同学补路段，再重新采集。

### 填错了怎么办

没关系。提交的是待审核记录，不会直接覆盖正式数据。告诉管理员驳回，重新提交即可。

## 11. 给评委的解释口径

我们不是依靠 GPS 自动判断无障碍条件，而是采用更可靠的现场采集流程：

```text
队友手机扫码进入采集页
选择预定义路段
现场填写适老字段
记录定位和备注作为证据
管理员审核
审核通过后更新正式路网
路线算法重新计算推荐结果
```

这样可以回答三个问题：

- 数据从哪里来：现场采集。
- 数据靠谱吗：有定位、备注、照片和审核。
- 非计算机队友能不能参与：只需要手机浏览器扫码填表。
