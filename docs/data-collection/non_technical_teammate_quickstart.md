# 非计算机队友手机采集使用说明

适用对象：不懂代码、不装环境、只负责现场采集的队友。  
一句话目标：像填问卷一样，用手机把每条路适不适合老人走记录下来。

## 1. 你需要准备什么

- 一部能上网的手机。
- 加入指定 Wi-Fi 或热点。
- 打开队长发的二维码。
- 到现场观察路段。

你不需要：

- 安装 Python。
- 安装数据库。
- 打开命令行。
- 理解代码。
- 修改任何系统配置。

## 2. 二维码怎么用

队长会发一个二维码。你用手机相机、微信或浏览器扫一扫。

扫码后会打开一个网页，这个网页就是助老地图系统。

如果打不开，先检查：

- 手机是不是连了指定 Wi-Fi 或热点。
- 电脑上的系统是不是已经启动。
- 二维码是不是今天新生成的。

## 3. 采集一条路段的流程

1. 打开网页。
2. 点击顶部“采集模式”。
3. 在“选择采集路段”里选择你当前所在的路段。
4. 点击“记录手机定位”。
5. 按现场情况填写字段。
6. 在备注里写清楚特殊情况。
7. 点击提交。
8. 提交后告诉审核同学，等审核通过。

## 4. 怎么判断自己有没有选对路段

选择路段时，看路段名称，不看数据库编号。

比如你人在三号门附近，就不要选择食堂入口路段。

如果你不确定当前是哪一段：

- 先问队友。
- 看系统里的路段名称。
- 仍然不确定就不要乱填，在备注里记录位置，回来让技术同学补路段。

## 5. 定位按钮有什么用

“记录手机定位”会保存你手机当前的大概经纬度。

它的作用是证明你大概在现场附近采集过。

但定位不是 100% 准确，校园里可能偏几米到十几米。所以最终以“你选择的路段 + 备注 + 照片 + 审核”为准。

如果定位失败，也可以继续提交，但备注里写：

```text
手机定位失败，已人工确认位于该路段现场。
```

## 6. 现场怎么分工最稳

建议两人一组：

- A 同学负责观察、测量、数台阶。
- B 同学负责手机填写、拍照、写备注。

每填完一条路，两个人互相确认：

```text
路段选对了吗？
有没有台阶？
有没有坡道？
有没有扶手？
轮椅能不能过？
备注写清楚了吗？
```

## 7. 不确定怎么填怎么办

不要硬猜。

可以先保守填写，并在备注里说明。

例如：

```text
坡度为目测估计，建议后续复测。
```

```text
路宽未带尺测量，为现场估计值。
```

```text
该路段人流随时间变化较大，当前采集时间为中午。
```

## 8. 哪些地方一定要写备注

遇到这些情况必须写备注：

- 有台阶。
- 有坡道但坡道很陡。
- 路面破损。
- 路很窄。
- 人车混行。
- 食堂、校门、医院附近人流大。
- 定位失败。
- 评分是估计值。
- 你不确定路段是否选对。

## 9. 提交后会发生什么

你提交的数据不会立刻改系统正式数据。

它会先进入“待审核记录”。

审核同学确认没有问题后，点击“通过并更新路段”，系统才会把你的采集结果写入正式路段数据，后面的路线推荐才会使用它。

如果填错了，不用慌。让审核同学驳回，然后重新提交。

## 10. 给队友的最短版本

如果现场时间紧，就记住这 8 步：

1. 连指定 Wi-Fi。
2. 扫二维码。
3. 进采集模式。
4. 选当前路段。
5. 点记录定位。
6. 填路宽、坡度、台阶、坡道、扶手、轮椅可通行等。
7. 写备注。
8. 提交，等审核。
