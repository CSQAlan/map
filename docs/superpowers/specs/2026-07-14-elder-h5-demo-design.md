# 老人端 H5 演示版设计

**日期：** 2026-07-14  
**项目：** 助老地图比赛版 MVP  
**子项目：** 老人端单页 H5 演示

## 1. 目标

本设计覆盖助老地图第一版前端演示页面。

目标是做一个可在比赛现场直接演示的 H5 单页应用，用来展示：

- 老人画像切换
- 起点终点选择
- MSR 适老路线推荐
- 最多 3 条候选路线
- 老人模式的大字、大按钮、低复杂度交互

## 2. 技术选择

前端采用：

- `Vue 3`
- `Vite`
- 原生 CSS

第一版不引入地图底图和复杂 UI 组件库。

原因：

- 当前重点是完成推荐闭环
- 不让地图渲染阻塞比赛演示
- 后续可替换为 Leaflet / 高德 / 腾讯地图等真实地图组件

## 3. 页面模式

单页内包含两个模式：

- 推荐模式
- 老人模式

顶部使用明显的分段切换控件，让用户可以在两个模式之间切换。

### 3.1 推荐模式

推荐模式服务于比赛演示。

包含：

- 老人画像选择
- 起点选择
- 终点选择
- 一键推荐按钮
- 路线候选列表
- 推荐理由
- 距离和预计时间

### 3.2 老人模式

老人模式服务于真实使用感。

包含：

- 当前推荐路线
- 大号开始导航按钮
- 大号重新推荐按钮
- 大号紧急求助按钮
- 简化的下一步路线说明

老人模式默认展示推荐模式选出的第一条路线。

## 4. 数据来源

前端调用后端接口：

`GET /api/routes/recommend`

请求参数：

- `start_name`
- `end_name`
- `mobility_type`

第一版使用固定候选：

- 起点：重庆师范大学三号门
- 终点：重庆师范大学校医院、重庆师范大学食堂
- 画像：独立出行型、辅助出行型、家属协同型

## 5. 展示方式

第一版暂不接真实地图。

路线展示采用：

- 路线卡片
- 路段流程线
- 关键指标
- 推荐理由

这样可以保证：

- 数据来自真实后端
- 展示稳定
- 比赛现场容易讲清楚

## 6. 交互流程

推荐模式流程：

1. 选择老人画像
2. 选择起点和终点
3. 点击推荐路线
4. 展示最多 3 条候选路线
5. 选择其中一条作为当前路线
6. 切换到老人模式查看简化展示

老人模式流程：

1. 查看当前推荐路线
2. 点击开始导航
3. 查看下一步提示
4. 可点击重新推荐
5. 可点击紧急求助

第一版的开始导航和紧急求助只做前端状态反馈，不接真实告警服务。

## 7. 视觉原则

整体风格要温和、清楚、有适老感，但不做医疗系统式的冰冷界面。

要求：

- 字号偏大
- 按钮明显
- 高对比
- 颜色不依赖单一色系
- 页面结构清楚
- 移动端优先

## 8. 错误处理

需要处理：

- 后端未启动
- 路线接口返回错误
- 没有候选路线
- 请求中

错误展示使用页面内提示，不弹复杂模态框。

## 9. 验收标准

完成后应满足：

- 能启动 Vue/Vite 前端
- 能调用后端路线推荐接口
- 能展示路线候选结果
- 能在推荐模式和老人模式之间切换
- 老人模式可显示当前路线
- 基础测试或构建通过
