# Dijkstra Route Planning Design

## Goal

Replace the route recommendation core from DFS full path enumeration to Dijkstra-based least-cost path search, while keeping the existing API response and frontend behavior unchanged.

## Scope

- Add a Dijkstra shortest-path function based on existing elder-profile segment costs.
- Add a lightweight Top-K candidate path function so the demo can still return up to three route options.
- Keep hard constraints such as wheelchair accessibility, width, slope, and no-ramp steps.
- Keep `recommend_routes(...)` response shape unchanged.
- Do not change database schema, frontend route rendering, or map data endpoints.

## Approach

Use segment cost as edge weight:

```text
edge weight = segment_cost(segment, mobility_type)
```

The first route is the lowest-cost Dijkstra path.

Additional candidates are generated by spur-path deviations from previous best paths:

1. Take the latest accepted path as the base path for the current expansion round.
2. For each node in that base path, keep the root path up to the spur node.
3. Temporarily block the next edge used by previous candidates with the same root.
4. Run Dijkstra from the spur node to the destination.
5. Combine root path and spur path, keep earlier candidates in a shared candidate pool, deduplicate by segment codes, and rank by total route score.

This is intentionally a small Yen-style Top-K implementation, enough for the current trial network and competition explanation.

## Error Handling

- If no path exists, return an empty list as before.
- If `start_node_code == end_node_code`, return an empty route list; the API already rejects same POI by name.
- Ignore disallowed segments before building graph.

## Testing

- Verify Dijkstra selects the lower-cost path instead of simply the first enumerated path.
- Verify blocked edges can force an alternative path.
- Verify `recommend_routes` still returns sorted top candidates.
- Verify wheelchair constraints still filter inaccessible paths.
- Run backend test suite and frontend build.
