# 师大苑真实地图与照片证据展示设计

## 1. 目标与范围

本阶段将现有师大苑 SVG 示意路网升级为高德真实底图，并把现场踩点照片作为路线推荐的可核验证据展示出来。首版只覆盖师大苑步行场景，不引入公交、驾车、骑行或其他试点区域。

交付目标：

- 地图默认并且仅聚焦师大苑试点范围。
- 17 条道路和 6 个 POI 叠加在高德底图上，并突出推荐路线与风险路段。
- 用户可从地图路段和路线详情两个入口查看同一组现场证据照片。
- 电脑和手机浏览器均可稳定演示。
- 高德地图不可用时，路线推荐和照片证据仍可使用。

## 2. 技术方案

前端直接使用高德地图 JavaScript API 2.0，不引入第三方 Vue 地图库。地图能力封装为独立 Vue 组件，避免地图实例生命周期、业务状态和页面表单继续堆积在 `App.vue` 中。

主要组件：

- `AmapRouteMap`：加载高德底图，管理师大苑边界、视野限制、道路覆盖物、POI 和选中状态。
- `EvidenceGallery`：展示当前路段的缩略图、无障碍指标、风险说明、数据来源与可信度。
- `PhotoLightbox`：展示较清晰照片，并支持手机端左右切换。
- `FallbackRouteMap`：高德加载失败时复用现有 SVG 路网，保证核心演示可继续。

地图 Key 与安全密钥通过 `frontend/.env.local` 注入。该文件必须被 Git 忽略。开发阶段使用高德允许的明文安全密钥方式；正式公开部署前改为服务端代理，并在高德控制台配置域名白名单。

## 3. 试点区域限制

新增 `pilot_area` 表，至少包含：

- `area_code`：稳定区域代码，师大苑固定为 `SHIDAYUAN`。
- `name`：区域名称。
- `boundary_geom`：WGS84 多边形边界。
- `center_geom`：WGS84 默认中心点。
- `min_zoom`、`max_zoom`：前端允许的缩放范围。
- `status`：区域启用状态。

`road_node`、`road_segment` 和 `poi_facility` 增加 `pilot_area_id` 外键。现有师大苑种子数据全部关联 `SHIDAYUAN`。地图和路线相关接口按已启用区域过滤，首版默认且只允许 `SHIDAYUAN`。

前端采用三层约束：

1. 使用师大苑中心点和小区级缩放初始化地图。
2. 使用高德 `setLimitBounds` 限制拖动范围。
3. 绘制师大苑边界，边界外区域视觉弱化。

高德公共底图仍可能请求边界附近的地图瓦片；“仅保留师大苑”指用户视野和本系统业务数据被限制在试点范围，而不是离线裁剪高德底图。

## 4. 坐标体系与校准

数据库中的业务几何继续以 WGS84（EPSG:4326）作为唯一标准存储，保留采集来源和原始坐标类型。高德展示接口输出 GCJ-02，并在响应顶层明确返回 `coordinate_system: GCJ02`，防止前端重复转换。

首版校准流程：

1. 将当前 WGS84 节点和边界转换为 GCJ-02。
2. 在高德底图上人工校准入口、荷塘、楼栋组团、主要路口等关键节点。
3. 由关键节点带动相邻路段几何调整。
4. 保留校准来源、时间和置信度，以便后续用手机轨迹替换近似几何。

坐标转换集中在后端服务中实现并单元测试，不允许前端组件各自转换。

## 5. 照片证据处理

56 张原始照片继续作为本地采集档案，不进入普通 Git 仓库。构建脚本从原图目录生成两类 WebP 文件：

- 缩略图：用于路线卡片和证据列表，目标为快速首屏加载。
- 展示图：用于大图查看，在用户点击后按需加载。

生成文件使用稳定照片 ID 命名，不依赖中文原始目录。照片清单记录照片 ID、原文件名、缩略图路径、展示图路径、拍摄点、说明和关联实体。数据库现有 `evidence_photo_refs` 保存照片 ID，API 将其解析为可访问的照片元数据。

后端只通过固定媒体目录和受控文件名提供生成后的图片，拒绝包含路径分隔符、上级目录或未知扩展名的请求。原图绝不通过接口暴露。

## 6. API 设计

### `GET /api/pilot-areas/SHIDAYUAN`

返回区域代码、名称、GCJ-02 中心点、边界、视野范围和缩放范围。未知或未启用区域返回 404。

### `GET /api/map-data/geojson?area_code=SHIDAYUAN&coordinate_system=GCJ02`

返回限定区域内的道路和 POI。每个要素补充：

- `evidence_photos`
- `data_confidence`
- `source_provider`
- `risk_summary`
- 无障碍核心指标

仅支持 `GCJ02` 和 `WGS84` 两种明确输出；默认地图展示使用 `GCJ02`。

### 照片资源

照片 URL 由 API 根据清单生成。请求只允许访问已登记的缩略图或展示图；缺失图片返回 404，前端显示统一占位图。

路线推荐响应中的路段继续使用同一 `segment_code`。前端以该代码关联地图要素、路线卡片和照片证据，避免复制关联逻辑。

## 7. 页面交互

地图初始展示师大苑边界、全部可用道路和 POI：

- 普通道路使用低饱和绿色或灰绿色。
- 台阶、裂缝等高风险路段使用橙红色虚线。
- 推荐路线使用更粗的高对比实线，未入选道路降低透明度。
- POI 使用适老化的大尺寸标记和清晰文字。

点击地图路段后选中该 `segment_code`，打开证据面板。点击路线详情中的路段卡片执行相同操作，并让地图定位到该路段。路线详情默认展示每段道路的一张代表照片，展开后展示全部照片。

大图查看包含照片、关联路段、风险说明和数据可信度。手机端支持左右切换，并保证关闭按钮、返回路线按钮和触控区域足够大。

地图提供“回到完整路线”和“定位当前位置”。当前位置只作为临时覆盖物展示，不自动修改道路或采集数据。

## 8. 加载与异常处理

- 高德脚本加载失败、鉴权失败或网络不可用时，切换到 `FallbackRouteMap` 并显示明确提示。
- 地图失败不阻断路线计算、画像切换、路线卡片和照片查看。
- 照片先加载缩略图；展示图仅在打开大图时加载。
- 单张照片失败时显示占位图，不隐藏对应路段。
- 缺少坐标的路段保留在路线详情中，但不创建地图覆盖物，并记录前端诊断信息。
- API 返回的坐标体系必须与请求一致；未知坐标体系返回 422。

## 9. 测试与验收

后端自动化测试覆盖：

- 试点区域过滤与禁用区域处理。
- WGS84 与 GCJ-02 坐标转换的固定样例和误差范围。
- GeoJSON 照片元数据与风险摘要。
- 非法照片 ID、目录穿越和缺失文件。
- 路线响应与地图要素使用一致的 `segment_code`。

前端验证覆盖：

- 缺少 Key、高德加载失败和照片加载失败时的降级界面。
- 地图路段与路线卡片双向联动。
- 路线切换后覆盖物样式和照片内容同步更新。
- 桌面浏览器与同一局域网手机浏览器的布局、触控和加载速度。

人工验收标准：

- 地图无法被拖离师大苑试点附近。
- 17 条道路和 6 个 POI 与真实底图基本对齐，关键节点位置可信。
- 推荐路线、风险道路和普通道路可在数秒内辨认。
- 任一路段从地图和路线卡片打开时展示同一组证据照片。
- 首屏不下载 56 张展示大图。

## 10. 本阶段不包含

- 高德步行路线替代本项目适老路径算法。
- 其他社区或重庆师范大学校园数据重新启用。
- 原图上传云存储、用户账号和照片审核后台。
- PWA 或原生 App 打包。
- 公交、驾车和骑行路线。
