# 多起终点与完整初始化 SQL 设计

## 背景与目标

当前路线后端已经能通过 `poi_facility.linked_node_code` 将 POI 解析为路网节点，但前端将起点列表硬编码为单个入口。因此，数据库中的多个重点 POI 虽然可以作为目的地，却没有作为起点提供给用户。

本次改造的目标是：所有状态为 `ACTIVE`、关联了有效路网节点且属于有效试点区域的 POI，都能同时作为路线起点和终点；同时生成一份可直接用于空 PostgreSQL/PostGIS 数据库的完整、可重复执行的初始化 SQL。

## 范围

本次包含：

- 从数据库动态查询可用路线端点。
- 起点和终点使用同一份 POI 候选数据。
- 禁止起点和终点选择同一个 POI。
- 完善现有种子初始化，使 Python 初始化与完整 SQL 初始化结果一致。
- 导出包含数据库结构和全部地图种子数据的完整 SQL。
- 增加初始化、端点查询和多起点路线推荐测试。

本次不包含用户自定义地点、任意经纬度吸附路网、跨试点区域路线或路线端点人工排序。

## 数据模型

采用动态派生方案，不新增 `is_start_point`、`is_end_point` 或端点配置表。

一个 POI 可作为路线端点，当且仅当：

1. `poi_facility.status = 'ACTIVE'`；
2. `poi_facility.linked_node_code` 非空；
3. 关联的 `road_node.osm_node_ref` 存在；
4. POI 与路网节点属于同一个试点区域；
5. 所属 `pilot_area.status = 'ACTIVE'`。

该规则使新增或更新 POI 后无需再维护独立的起点标记，避免起点和终点能力不一致。

## 接口与应用数据流

新增语义明确的路线端点查询接口，返回端点的 POI ID、名称、类型和关联节点编码。端点查询和路线推荐通过同一个后端查询函数执行端点筛选，避免两套条件产生差异。现有地图 POI 接口保持兼容，不承担路线端点契约。

前端启动时读取端点列表，起点和终点选择器共用该列表。默认起点优先沿用师大苑入口；默认终点选择第一个不同于默认起点的端点。用户修改任一选择时，另一选择器不得提供相同值。提交路线推荐前仍由前后端同时校验起点和终点不同。

路线推荐继续使用 POI 名称作为现有请求参数，以控制本次改造范围；数据库查询必须限定试点区域并验证关联节点。后续如需处理重名 POI，可单独迁移为 POI ID 参数。

## 初始化与完整 SQL

初始化顺序为：

1. 创建 PostGIS 扩展和公共更新时间函数；
2. 创建表、约束、索引和触发器；
3. 写入试点区域；
4. 写入路网节点；
5. 写入路段；
6. 写入 POI；
7. 执行数据一致性校验。

数据写入使用稳定业务键和 `INSERT ... ON CONFLICT DO UPDATE`，保证脚本可重复执行且不会产生重复数据。稳定业务键分别为：

- 试点区域：`area_code`；
- 路网节点：`osm_node_ref`；
- 路段：`segment_code`；
- POI：使用 `(pilot_area_id, name, poi_type)` 复合唯一约束进行幂等匹配。

POI 几何坐标优先取关联路网节点的 `geom`，只有在导入兼容数据且不存在节点坐标时才允许使用显式经纬度。完整 SQL 文件应包含事务边界，关键校验失败时整体回滚。

导出的 SQL 面向空 PostgreSQL 16 + PostGIS 数据库，同时允许对已初始化数据库重复执行。SQL 中包含当前项目的全部有效试点区域、节点、路段和 POI 种子数据，不包含本地未纳入版本控制的原始照片文件。

## 一致性校验与错误处理

初始化完成前执行以下校验：

- 活跃 POI 的 `linked_node_code` 是否能解析到同区域路网节点；
- 活跃路段的起止节点是否存在且属于同区域；
- 是否存在孤立的路线端点；
- 每个活跃试点区域是否至少有两个可用端点；
- 当前路网中是否存在从各端点出发完全不可达的端点。

结构或关键关联异常应抛出错误并回滚事务。由于老人画像的无障碍硬约束可能使特定端点组合不可达，这类情况不应阻止数据库初始化，而应由路线接口返回清晰的“无可达路线”结果及规避路段说明。

## 测试与验收

至少覆盖以下场景：

- 在空数据库执行完整 SQL 后，结构和种子数据齐全；
- 连续执行两次完整 SQL，记录数量不增加且数据一致；
- Python 初始化与完整 SQL 初始化产生相同的业务键集合和关键字段；
- 所有符合端点条件的 POI 都出现在端点接口中；
- 无关联节点、跨区域关联或非活跃 POI 不出现在端点接口中；
- 任一可用 POI 均可作为起点请求路线；
- 起点与终点相同时，前端阻止提交且后端返回 `400`；
- 不同老人画像下不可达路线返回预期错误，不破坏其他端点组合；
- 前端构建与后端测试全部通过。

验收标准是：页面不再只有单一起点；数据库中所有有效关联 POI 均能被选择为起点或终点；完整初始化 SQL 可在空库成功执行并可安全重复执行。
